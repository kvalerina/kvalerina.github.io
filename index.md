# <span style="color:#1773a0; font-size:1em;">Description of the App</span> <br>
<p style="font-size: 1.2em; text-align: justify;">
The Event Planning App is a mobile application designed to help users manage and organize their events efficiently. This app has evolved significantly since it's original implementation, through enhancements focused on improving functionality, security, and user experience. Originally written in Java, the app has been converted to Kotlin, leveraging modern language features like null safety, concise syntax, and improved maintainability. The app allows users to perform essential tasks such as creating, updating, and deleting events, with data stored securly in Firebase. <br>
</p>

<p style="font-size: 1.2em; text-align: justify;">
Enhancements have been made to ensure the app not only functions as expected but also adheres to industry standards for software design, database management, and user experience. The app has been improved by transitioning from a single, ongoing list of events to separate daily and monthly views, providing users with more organized and intuitive access to their schedules. Integrating Firebase for real-time data synchronization and adding advanced features like event prioritization and filtering, will take the app’s functionality to the next level. This app showcases my ability to develop and enhance mobile applications while demonstrating skills in software engineering, UI/UX design, and database management. It serves as a reflection of my growth as a developer and my dedication to creating user-centered, secure, and maintainable solutions.

# <span style="color:#1773a0; font-size:1em;">Enhancement One: Software Design and Engineering</span> <br>
<p style="font-size: 1.2em; text-align: justify;">
I included my Event Planning app in my ePortfolio because it really highlights the progress I've made as a developer. For this enhancement, I transitioned the code from Java to Kotlin, which made the app more readable, maintainable, and in line with current development practices. Converting the code allowed me to take advantage of Kotlin’s modern features like null safety and concise syntax. I also focused on improving the app’s security and user experience by adding robust validation for user credentials and event details, along with switching to EncryptedSharedPreferences to ensure better protection of sensitive information. The app’s interface was also improved with snackbars for clearer feedback, and I added error handling to make it more reliable.
<br><br>
Throughout the process, I faced some challenges, especially when converting the app to Kotlin and figuring out how to implement the best validation checks and exception handling. I had to rethink how certain parts of the app were structured and how to balance usability with security. Even though there were some hurdles, I was able to improve the app’s functionality, security, and overall user experience. This project helped me meet key course outcomes related to software design, security, and modern development practices, and it reinforced the importance of making an app both secure and user-friendly. <br>

# <span style="color:#1773a0; font-size:1em;">Enhancement Three: Databases <br>
<p style="font-size: 1.2em; text-align: justify;">
I chose to include the Event Planning app in my ePortfolio because it highlights my ability to work with cloud-based technologies, specifically Firebase, which is essential in today’s software development industry. For Enhancement 3, I migrated the app’s database from SQLite to Firebase, which significantly improved the app’s scalability, real-time updates, and accessibility across devices. The integration of Firebase Firestore for data storage and Firebase Authentication for secure login made the app more secure and user-friendly. This enhancement not only improved the app’s functionality but also demonstrated my skills in modern database management and secure data handling, both of which are essential for building reliable, scalable apps.
<br><br>
During the process of migrating the database, I faced several challenges, particularly with effectively organizing the data and ensuring the app continued to function properly during the transfer. One of the biggest challenges was integrating Firebase Authentication, which required some troubleshooting to ensure secure user login. Additionally, I had to make sure the real-time data syncing didn’t cause delays or affect performance. Overcoming these problems gave me deeper insights into cloud storage and Firebase’s capabilities. The experience helped me meet my goals of improving the app’s database management and functionality, and I look forward to exploring more advanced Firebase features in future projects.
